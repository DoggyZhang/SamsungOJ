问题 A: ★Map Completion

时间限制: 10 Sec  内存限制: 256 MB  Special Judge
提交: 1037  解决: 258
[提交][状态][讨论版]

题目描述


Restrictions

Execution time: within 3 sec for maximum 50 test cases

Memory: Maximum 256MB available for heap, global and stack combined (note: Maximum 1MB can be used for stack)


[Cautions for Coding]

1. Test takers who will be using C or C++ to write their answer, please select C++ as for the Language before submit the answer.

2. Consists of the Main part and User Code part.

       A. Main Part: Cannot be edited. Codes just as submitted will be used for grading.

       B. User Code Part: Code which the test taker must actually write.

       When submitting the answer, not only library functions within the code but also standard input/output functions cannot be included.

       (Note: Only the malloc.h is available in case of C/C++)

       C. In the server, main part and user code part will be split to separate files and compiled.

       (main.cpp/solition.cpp or Solution.java/UserSolution.java)

3. Cautions When Programming on Local PC

       A. 2 files must be generated.

       (main.cpp/solition.cpp or Solution.java/UserSolution.java)

       B. Please copy the Main part into main.cpp or Solution.java

       C. To user the sample_input.txt, uncomment the code (commented) which uses the file as the standard input within the Main part code.

       D. After writing the User code part, before submitting, delete library functions for debugging and all standard input/output functions or comment.

4. Since not all constrains are specified in this document, analyze the given code for further information.

5. Given codes can differ depending on the programming language. Analyze codes according to your language selection.


[Description]

There is an NxN size terrain.

We are trying to make a map of the terrain as show in [Fig.1].


[Fig.1]


The number on each cell denotes topographic information of the terrain.

The topographical information of two outermost borderline cells of the terrain is known, but no other terrain information is known.


[Fig.2]


A satellite scans 4x4 size terrain information as it passes over the terrain.

We are trying to complete the map by using this information.

However, you do not know the exact location of the scanned terrain.

Only the information that the top leftmost position of the scanned terrain information is located in an even-numbered row and an even-numbered column is known. (Row and column starts from 0.)

[Fig.3] 


Additionally, the scanned information does not deviate from the given terrain.

When the information about two outermost borderline cells of the terrain is given, implement an API that completes N x N terrain map from the scanned information. 

It is favorable to call as little as possible the randomscan() function, which carries the scanned terrain information.

Implement each function the API explanation below.


※ The function signature below is based on C/C++. As for Java, refer to the provided Solution.java and UserSolution.java.

void reconstruct (int N, char map[][])

This is a function that completes the map.


When this function is called, the topographical information of two outermost borderline cells is stored in the corresponding location in the map array. In other locations, 0 values are stored.


When the function ends, the completed map must be stored in the map array.


Parameters

N: the length of a side of the map ( 8 <= N <= 64, multiple of 4)

map: The map array to complete (1 <= map[][] <= 15)

 

Below is the explanation of API which is provided in the Main part.

void randomscan (char zone[][])

This is a function that scans the information of terrain with a size of 4x4 at a random location.


The scanned information is stored in the zone.

The top leftmost position of the scanned information is always an even-numbered column and even-numbered row.


Parameters

zone: An array in which the scanned terrain information is stored.

For more detailed information, analyze the Main part.



[Constraints]

1. The length of one side of the terrain is a multiple of four, greater than or equal to 8 and less than or equal to 64.

2. The terrain information is an integer greater than or equal to 1 and less than or equal to 15.

3. randomscan() function is called at most 100,000 times.

4. The less you call randomscan() function, the more favorable your answer will be.

5. In a given test case, if the number of pieces of terrain information matching the scanned information is more than or equal to 8, and there is no different information when the terrain information and the scanned information is compared, then it is guaranteed that it is the scanned area.


输入

//main.cpp 


#ifndef _CRT_SECURE_NO_WARNINGS
#define _CRT_SECURE_NO_WARNINGS
#endif

#include <stdio.h>

#define MAX_MAP_SIZE		64
#define ZONE_SIZE			4

#define MAX_SCAN_COUNT		100000

extern void reconstruct(int N, char map[MAX_MAP_SIZE][MAX_MAP_SIZE]);

static char map[MAX_MAP_SIZE][MAX_MAP_SIZE];
static int N;
static int seed;
static int scancount;

static int pseudo_rand() {
	seed = seed * 214013 + 2531011;
	return (seed >> 16) & 0x7fff;
}

void randomscan(char zone[ZONE_SIZE][ZONE_SIZE]) {
	if (scancount >= MAX_SCAN_COUNT)
		return;

	int x0 = (pseudo_rand() % ((N - ZONE_SIZE) / 2 + 1)) * 2;  // x0 = 0, 2, 4, ..., N - 4
	int y0 = (pseudo_rand() % ((N - ZONE_SIZE) / 2 + 1)) * 2;  // y0 = 0, 2, 4, ..., N - 4
	
	for (int y = 0; y < ZONE_SIZE; ++y)
	for (int x = 0; x < ZONE_SIZE; ++x)
		zone[y][x] = map[y + y0][x + x0];
	
	++scancount;
}

static void makemap() {	
	for (int y = 0; y < N; ++y)
	for (int x = 0; x < N; ++x)
		map[y][x] = pseudo_rand() % 15 + 1;
}

static bool compare(char map1[MAX_MAP_SIZE][MAX_MAP_SIZE], char map2[MAX_MAP_SIZE][MAX_MAP_SIZE]) {
	for (int y = 0; y < N; ++y)
	for (int x = 0; x < N; ++x)
		if (map1[y][x] != map2[y][x])
			return false;
	
	return true;
}

int main() {
    int TC, totalscore, totalscan;

    //freopen("sample_input.txt", "r", stdin);

	setbuf(stdout, NULL);
    scanf("%d", &TC);

    totalscore = totalscan = 0;
    for (int testcase = 1; testcase <= TC; ++testcase) {
		char guess[MAX_MAP_SIZE][MAX_MAP_SIZE];
		int  min_scan_count;
		
		scanf("%d %d %d", &N, &seed, &min_scan_count);

        makemap();
		
		scancount = 0;
		
		for (int y = 0; y < N; ++y)
		for (int x = 0; x < N; ++x)
			guess[y][x] = 0;
		
		for (int k = 0; k < N; ++k) {
			guess[k][0] = map[k][0];
			guess[0][k] = map[0][k];
			guess[k][1] = map[k][1];
			guess[1][k] = map[1][k];
			guess[k][N - 1] = map[k][N - 1];
			guess[N - 1][k] = map[N - 1][k];
			guess[k][N - 2] = map[k][N - 2];
			guess[N - 2][k] = map[N - 2][k];
		}
		
		reconstruct(N, guess);
		
		if (!compare(map, guess))
			scancount = MAX_SCAN_COUNT;
		totalscan += scancount;
		
		int score;
		if (TC == 5) { // sample testcase
			score = scancount < MAX_SCAN_COUNT ? 100 : 0;
			totalscore += score;
		} else { // eval testcase
			score = scancount <= min_scan_count + (min_scan_count / 2) ? 100 : 0;
			totalscore += score;
		}
		
		printf("#%d %d\n", testcase, score);
    }

	if (totalscan > MAX_SCAN_COUNT) totalscan = MAX_SCAN_COUNT;
    printf("total score = %d, total scan = %d\n", totalscore / TC, totalscan);
    return 0;
}
