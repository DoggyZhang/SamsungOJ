问题 A: ★Typo Correction System

时间限制: 10 Sec  内存限制: 256 MB
提交: 229  解决: 120
[提交][状态][讨论版]

题目描述

In many search engines,they provide a function that show the corrrected word when the user inputs a typo as shown in [Fig.1]

Typo Correction System updates the Typo Correction DB and returns the results in real time based the user's search info.

The users of the system will attempt to search within 10s,if they think that they have searched a typo.

If the re-searched word within 10s corresponds to the type of typo defined in this system,it can be registered in the Typo Correction.

This system defines the type if typo as shown in [Table 1] below.  

 

                                        [Table 1]

The word can be registered only when it applies to just one of three Addition,Substitution,Deletion which are defined in [Table 1].
The word can't be registered if two ot three cases of the above overlaps.

If the same typo,correction set appears on 3 different users,then register the word to Typo Correction DB.

After that,if a word that is registered in the Typo Correction DB is searched the corrected is returned.

There may be two or more correct word for one typo.
eg.geometri(typo) - geometry(correct word), geometri(typo) - geometric(correct word)

 

Implement a Typo Correction System that operates as described in the above.

 

The following are the APIs to be implemented.
# In terms of Java,char is in 2 byte representation,but only uses the low byte.

void init(int N)
This function is called once at the beginning of each test case for initialization.

Parameters
N:the user number (5 <= N <= 2000)

int search(int mId, int searchTimestamp, char searchWord[11], char correctWord[5][11])
User of mId searches at searchTimestamp time with searchWord.
If the typo is corrected,the correct word is substituted and returned to correctWord.

Parameters
  mId:id of the search user(1 <= mId <= N)
  searchTimestamp:search time.Unit is seconds(1 <= searchTimestamp <= 50000)
  searchword[11]:search word(4 <= length of the word <= 10)
  correctWord[5][11]:corrected word(4 <= length of the word <= 10)
Return
  Return the number of corrected words.

[Constraints]
  1.The user number, N, is an integer greater than or equal to 5 and less than or equal to 2000.(5 <= N <= 2000)
  2.the number of words searched per test case is less than 4000,including typos.
  3.It is guaranteed that one user does not search more than three times in 20s.
  4.It is guaranteed that there will be less than or equal to five correct words for one typo.
  5.The searchWord in search function consists only of lowercase alphabet characters,and the length is 4 or more and 10 or less.
  6.The search function is called in the increasing order of searchTimestamp,and searchTimestamp is not overlapped.
  7.The search call count is less than or equal to 50000.
